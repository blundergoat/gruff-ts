# Changelog

## v0.2.0 - 2026-05-28

Cross-format Pillars table, per-command gating defaults, JSON schema bumps (`v1` → `v2`), exported-doc rule split, a rule-precision sweep, config schema-versioning, and a code-review sweep against PR #4 feedback. Cut as a pre-1.0 minor bump because the cumulative rule-emission changes (new exported-doc surface, tightened naming/test-quality exemptions, schemaVersion requirement) materially shift findings for downstream consumers.

- **Per-command gating + schemaVersion contract** - `analyse` and `summary` default `--fail-on` flips from `error` to `advisory`; CI gates using the defaults go from "fail on errors only" to "fail on anything." New `minimumSeverity:` block in `.gruff-ts.yaml` pins per-command thresholds (`analyse / summary / report`) with precedence CLI flag > config > binary default; `dashboard` is rejected as a key because it has no `--fail-on` flag. `.gruff-ts.yaml` now requires `schemaVersion: gruff-ts.config.v0.1` at the top - pre-1.0 hard break, no transitional shim. Run `npx gruff-ts init --force` to migrate; the regenerated config preserves `paths.ignore` and `minimumSeverity`. (ADR-004, `src/config.ts`, `src/cli-program.ts`)
- **Cross-format Pillars table** - Canonical 7-column table (`Pillar | Grade | Score | Findings | Advisory | Warning | Error`) renders byte-aligned across `text`, `json`, `html`, and `markdown`, sorted `findings DESC` then `pillar ASC`. Clean pillars render as `A/100` rows so the column layout stays stable run-to-run; cross-port consumers can diff byte-for-byte. (`src/pillar-summary.ts`, `src/report-renderers.ts`, `src/report-html.ts`)
- **JSON schema bumps `v1` → `v2`** - `summary --format=json` emits `gruff.summary.v2`; the flat `{pillar, count}` per-pillar array is replaced with a richer shape carrying `grade`, `score`, `penalty`, `applicable`, `findings`, `advisory`, `warning`, `error`. `analyse` / `report --format=json` emit `gruff.analysis.v2`; `score.pillars[]` entries gain a `penalty: number` field, and the SARIF `runs[0].properties.gruffSchemaVersion` plus the JSON `schemaVersion` tag move in lockstep. Downstream consumers must gate on the `schemaVersion` field before decoding.
- **Public-API doc gate** - `docs.missing-function-doc` (advisory) is split into `docs.missing-exported-function-doc` (**warning**) and `docs.missing-internal-function-doc` (advisory). The warning variant fires on `export function`, `export const f = () => ...`, `export default function`, and locally-declared functions re-exported via `export { foo }` or `export default foo`; `export { foo } from "./other"` re-export-from clauses are correctly skipped via negative lookahead so a same-named local stays internal. Class methods of exported classes are intentionally treated as internal (advisory) by this release; enclosing-class scope tracking is deferred. Any `.gruff-ts.yaml` override targeting the old rule ID must be updated. (M04, `src/blocks.ts`, `src/doc-rules.ts`)
- **Summary readability + rule explain mode** - Per-severity grade breakdown under the composite (`Errors: A (0)`, `Warnings: D (276)`, `Advisory: F (1367)`) tells you whether an F composite is driven by errors or by a long tail of advisories; three grade pills in HTML. `summary` Top-N rule rows now show per-severity split and one-line descriptions: `- <ruleId>: <total> (<err> err / <warn> warn / <adv> adv) - <description>`; a `Tip:` line points at `summary --top=20` when `analyse` produces 50+ findings. `gruff-ts list-rules <ruleId>` accepts a positional argument and prints per-rule detail (text or JSON) with `Config keys:`, `Options:`, and `Allowlists:` blocks. Every tunable rule's `remediation` text now names its override knob (`rules.<id>.threshold`, `rules.<id>.severity`, allowlist keys) so the configuration surface is one greppable string away from any finding. (M05-M08)
- **Rule-precision sweep** - `naming.short-variable` exempts single-character `for (const X of Y)` bindings whose body is ≤ 10 lines (and only those; classic `for (let i = 0; ...)` and `for (const k in obj)` stay covered). `test-quality.loop-in-test` recognises fixture loops (literal-array iterable or `Object.entries|keys|values(literal)`, no conditional bodies, last meaningful statement is an assertion) and suppresses the finding for them. `waste.swallowed-catch` accepts six additional rationale comments: `/* ignore */`, `/* ignored */`, `/* cleanup */`, `/* teardown */`, `/* noop */`, `/* no-op */`. `waste.unreachable-code` no longer fires on the unconditional return after a braceless `if`/`while`/`for` with a multi-line predicate. Nine imperative-flag verbs added to default `booleanPrefixes` (`check`, `skip`, `enable`, `allow`, `include`, `exclude`, `with`, `without`, `omit`); `fn` and `cb` added to default `acceptedAbbreviations`. (M01-M03)
- **Severity-only rule overrides accepted** - `rules.<id>.severity: warning` now loads without requiring `threshold`, fixing the validator that previously blocked any severity tweak on rules with no `threshold` knob (security, naming, docs, etc.) even though `gruff-ts list-rules` advertised the override. Each field is independently optional. (`src/config.ts:assertRuleThresholdConfig`)
- **Graceful CLI config errors** - Malformed `.gruff-ts.yaml`, missing `--config <path>`, and syntax errors in `.gruff.json` produce a formatted `gruff-ts: config error\n  ...\nSuggested fix: ...` message and exit code 2; raw `ENOENT` / `SyntaxError` no longer leak through as Node stack traces. `init --force` over pre-schemaVersion configs preserves `paths.ignore` and `minimumSeverity` via a new permissive reader (`src/config-preservation.ts`), so existing curated entries survive the schemaVersion migration. (`src/cli-program.ts:runWithConfigErrorHandling`, `src/config.ts:parseConfigFile`, `src/init-config.ts`)
- **Release tooling** - `scripts/bump-version.sh --check` verifies `CHANGELOG.md`'s most-recent version heading agrees with `package.json` / `package-lock.json` / `src/constants.ts`; the preflight gate's `Version consistency` step uses it. The awk regex skips non-version headings (e.g. `## [Unreleased]`-style entries from third-party Keep-a-Changelog projects).
- **Regression coverage** - FP-#12 through FP-#45 added to `src/false-positive-fixes.test.ts`, covering every rule-precision change above plus negative regression guards. The PR #4 review sweep added FP-#33b through FP-#33e for re-export detection, the C-style for / for-in regression cases under `naming-rules.test.ts`, the pre-schemaVersion preservation regression in `src/init-config.test.ts`, and the dedicated `src/config-preservation.test.ts` suite. M06 / M08 land their tests in `src/rule-catalogue.test.ts` and `src/cli-surfaces.test.ts` respectively; M10-M12 add parser-rejection tests in `src/project-config-rules.test.ts`. M09 (informational tier) was deleted as a no-signal milestone.
- **Cross-port harmony** - `minimumSeverity:` dimension aligns with the gruff family (`gruff-go` / `gruff-rs` / `gruff-py` / `gruff-php` should track). Off-switch value is `none` (not `never`) per the noun-position re-evaluation; the verb-shaped `failOn: never` was the prior cross-port draft, the noun-shaped `minimumSeverity: none` is the family-wide decision.

## v0.1.1 - 2026-05-24

Onboarding flow, machine-readable summaries, and a `waste` → `maintainability` pillar rename. Catalogue is now 119 rules across 11 pillars.

- **Pillar rename `waste` → `maintainability`** - The pillar tag attached to every finding from a `waste.*` rule changed; rule IDs themselves keep their `waste.` prefix (so `waste.console-log` still fires as `waste.console-log`, only its pillar attribute moves). Text/JSON/SARIF parsers, dashboard nav, and the `Pillar` TypeScript union in `src/types.ts` must update.
- **Removed rules and config keys** - `docs.todo-density` and `naming.abbreviation` (both opt-in advisories) removed, along with the now-unused `allowlists.abbreviationDenylist` config key; stale config entries are silently ignored so existing `.gruff-ts.yaml` files keep loading. Consumers relying on either rule for CI gating lose that signal.
- **`gruff-ts init`** - Writes the default `.gruff-ts.yaml`, refusing to overwrite any supported config name without `--force` (which preserves existing `paths.ignore` entries). `analyse`, `summary`, `report`, and `dashboard` auto-prompt for `init` when no config is found, gated by `--no-interaction`, `--silent`/`--quiet`, and TTY checks so it never fires in CI.
- **Machine-readable summaries** - `summary --format=json` emits the new `gruff.summary.v1` schema; `summary --top <n>` controls digest size; a baseline status line surfaces source and suppression count. `dashboard --project-root <path>` sets the default scan root.
- **Docs index + audit gate** - New `docs/README.md` index plus `ci-integration.md`, `dashboard.md`, `output-formats.md`, `rules.md`; CI now runs `npm audit --audit-level=moderate`.
- **CLI input rejection + async exits** - `summary` and `list-rules` reject unsupported `--format` values via Commander instead of coercing to `text`; CLI switched to `parseAsync` so async handler rejections exit with the correct code.
- **Top-offender + flag-knowledge fixes** - `summary --top <n>` controls file-offender count beyond 10. `gruff.analysis.v1` `score.topOffenders` is now the full sorted file list (was capped at 10); HTML report and `--format=hotspot` still cap their own output at 10 rows. `init --force` help text now reflects that it also overrides the refusal triggered by non-canonical configs (`.gruff.yaml`/`.yml`/`.json`). `--top` added to the known-CLI-flag list so comments documenting it no longer trip `docs.stale-comment`.
- **Doc filename normalisation** - `CONFIGURATION.md` → `configuration.md`, `RELEASING.md` → `releasing.md`, `REPORTS_AND_CI.md` → `reports-and-ci.md`.

## v0.1.0 - 2026-05-23

Initial public release of the `@blundergoat/gruff-ts` npm package.

- **TypeScript/JavaScript code-quality scanner** - 121 rules across 11 pillars: complexity, dead-code, design, documentation, maintainability, modernisation, naming, security, sensitive-data, size, test-quality.
- **CLI commands** - `analyse`, `summary`, `report`, `list-rules`, `dashboard`, `completion`.
- **Output formats** - `text`, `json`, `html`, `markdown`, `github`, `hotspot`, `sarif`. Stable schemas: `gruff.analysis.v1`, `gruff.baseline.v1`, `gruff.hotspot.v1`.
- **Baselines and diff scans** - Stable per-finding fingerprints; `--diff` for changed-file scans against the prior baseline. Local dashboard on `127.0.0.1:8767`.
