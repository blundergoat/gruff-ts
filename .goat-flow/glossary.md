# Glossary

Project-specific terms used across `src/*.ts`, the public schemas, and downstream tooling.

- **Pillar** — One of 11 quality dimensions a finding belongs to: `size`, `complexity`, `dead-code`, `waste`, `naming`, `documentation`, `modernisation`, `security`, `sensitive-data`, `test-quality`, `design`. Defined as the `Pillar` union in `src/types.ts` (search: `export type Pillar`).
- **Severity** — `advisory` | `warning` | `error`. Drives `--fail-on` exit-code logic (`exitFor` in `src/scoring.ts`, search: `function exitFor`) and pillar score penalty in `severityPenalty` (search: `function severityPenalty`).
- **Confidence** — `low` | `medium` | `high`. Recorded on each `Finding` so consumers can filter; not currently used in scoring.
- **Finding** — One reported issue. Shape declared in `src/types.ts` (search: `export interface Finding`). Required: `ruleId`, `message`, `filePath`, `severity`, `pillar`, `secondaryPillars`, `tier`, `confidence`, `metadata`, `fingerprint`. Optional: `line`, `endLine`, `column`, `symbol`, `remediation`. Naming findings carry `metadata.surface` such as `"declaration"`, `"parameter"`, `"destructure"`, or `"interface-field"` to identify where the rule fired.
- **Fingerprint** — 16-char sha256 prefix of `ruleId\0filePath\0line\0symbol`, computed in `makeFinding` (`src/findings.ts`, search: `function makeFinding`). Stable across runs and the dedupe / baseline-match key.
- **Rule ID** — Dot-namespaced string of the form `<namespace>.<rule-slug>` (e.g., `size.file-length`, `security.eval-call`, `docs.todo-density`, `naming.short-variable`). Used as `Finding.ruleId`, in baseline entries, and in config `rules` overrides. Documentation rules use `docs.*` while their pillar is `documentation`.
- **Gruff config** — Optional project config loaded from `.gruff-ts.yaml` or an explicit YAML `--config` path. Shared keys are `paths.ignore`, `allowlists.acceptedAbbreviations`, `allowlists.secretPreviews`, `allowlists.bannedGenericNames`, `allowlists.booleanPrefixes`, `allowlists.hungarianPrefixes`, `allowlists.placeholderNames`, `allowlists.abbreviationDenylist`, `allowlists.negativeBooleanAllowed`, `allowlists.knownAcronyms`, and `rules`. `selection` is not implemented in `gruff-ts` yet, so checked-in config should not include it.
- **Tier** — Currently fixed at `"v0.1"` on every finding; placeholder for future rule-set versioning.
- **Baseline** — A `gruff.baseline.v1` JSON file (default path `gruff-baseline.json`) that suppresses matching findings on subsequent runs. Match key is `(fingerprint, ruleId, filePath)`. Written by `writeBaseline` (`src/baseline.ts`, search: `function writeBaseline`), applied by `applyBaseline` (search: `function applyBaseline`).
- **Diagnostic** — Non-finding run-time issue: `parse-error` (unbalanced braces/parens/brackets), `read-error`, `missing-path`, `history-error`. Diagnostics force exit code 2 regardless of `--fail-on`.
- **Function block** — A regex-matched function unit returned by `functionBlocks` (`src/blocks.ts`, search: `export function functionBlocks`). Carries `name`, `params`, `startLine`, `lineCount`, `body`, `isPublic`, `isTest`. Powers per-function rules (size, complexity, naming, doc, test-quality).
- **Schema versions** — Three public string contracts: `gruff.analysis.v1` (full report), `gruff.baseline.v1` (suppression file), `gruff.hotspot.v1` (trimmed top-offenders payload). Bump only on breaking changes.
- **Hotspot output** — Compact format emitted by `--format hotspot`: `{ schemaVersion, tool, score, files: topOffenders }`. Designed for dashboards that want one number + worst files per project.
- **Default-ignored directory** — Hardcoded list in `isDefaultIgnoredDir` (`src/discovery.ts`, search: `function isDefaultIgnoredDir`): `.git`, `.hg`, `.svn`, `.idea`, `.vscode`, `build`, `cache`, `coverage`, `dist`, `generated`, `node_modules`, `target`, `tmp`, `vendor`. Bypassed by `--include-ignored`.
- **Accepted abbreviation** — Two-char-or-shorter variable names normally tripped by `naming.short-variable` are exempted if listed here. Defaults: `id`, `db`, `fs`, `io`, `ui`, `tx`, `rx` (plus loop counters `i`, `j`, `k`). Extendable via config `allowlists.acceptedAbbreviations`. Companion naming-blacklist keys with replace semantics (supply `[]` to empty): `allowlists.bannedGenericNames` (defaults `process, handle, doit, run, execute, manage` — drives `naming.generic-function`), `allowlists.booleanPrefixes` (defaults `is, has, can, should, does, did, was, will, may, in, scan, supports, requires` — drives `naming.boolean-prefix`), `allowlists.hungarianPrefixes` (defaults `str, obj, arr, bool, int, num` — drives `naming.hungarian-notation`), `allowlists.placeholderNames` (defaults `foo, bar, baz, tmp, temp, thing, stuff, data, value, item` — drives `naming.identifier-quality` generic branch and `naming.generic-parameter` placeholder check; numbered-suffix branch of identifier-quality is unaffected), `allowlists.abbreviationDenylist` (defaults `ctx, pkg, opts, fn, idx, cb` — drives opt-in `naming.abbreviation`, case-insensitive), `allowlists.negativeBooleanAllowed` (defaults `nostore, nofollow, noreferrer, noscript, noindex` — silences `naming.negative-boolean` for domain-legitimate negations), and `allowlists.knownAcronyms` (defaults `url, http, https, id, xml, json, html, css, api, sql, db, io, ui, uuid, ip, tcp, udp, ast, cli, npm` — drives `naming.acronym-case`). Pillar has 11 rules; `naming.inconsistent-casing` and `naming.acronym-case` build a per-file inventory of variable-style declarations (const/let/var, parameters, interface fields) and skip type/class/function names to avoid the PascalCase-type vs camelCase-instance idiomatic conflict.
- **Secret preview** — Redacted, truncated form of a sensitive-data match (`<first4>...<last4> (redacted, N chars)`) used for both finding metadata and the `allowlists.secretPreviews` allowlist. Computed by `redact` (`src/sensitive-data-rules.ts`, search: `function redact`).
